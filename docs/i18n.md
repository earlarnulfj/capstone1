I18n System Overview

Purpose
- Centralize system messages and support multiple languages.
- Enable dynamic switching and scalable additions.

Files
- `js/i18n.js`: runtime i18n module with English fallback and a minimal selector UI.
- `config/translations.json`: master translations configuration.
- `js/variation_sync.js`: uses i18n for variation-related messages.
- `scripts/validate_translations.php`: checks completeness vs English.

Usage
- Add languages: extend `languages` and `translations` in `config/translations.json`.
- Switch language: use the selector injected at top-right or call `I18N.setLanguage('xx')`.
- Translate messages: reference keys like `variations.applied_updates` and supply params.

Fallback & Validation
- Missing keys fall back to English. If not found, the key string is used.
- Run `php scripts/validate_translations.php` before deployment to ensure completeness.

Performance
- Config is loaded once on DOM ready and cached in memory.
- Message resolution is O(1) with simple string interpolation.

Maintenance
- Keep English dictionary exhaustive; other languages should mirror keys.
- Prefer short, clear messages. Use placeholders: `{count}`, `{ms}`, `{status}`.
